# App Icon Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a dock/taskbar application icon (recovery arrow + play triangle, neutral slate palette) to TeslaCam Recovery.

**Architecture:** Write the source SVG at `build/icon.svg`, convert it to a 1024×1024 PNG at `build/icon.png` using `sharp`, and commit both. `electron-builder` auto-discovers `build/icon.png` and generates platform-specific icon formats (`.icns`, `.ico`) during `npm run build`.

**Tech Stack:** SVG, Node.js, `sharp` (npm dev dep), `electron-builder`

### Task 2: Convert SVG to PNG

**Files:**
- Create: `build/icon.png` (generated, also committed)

- [ ] **Step 1: Install sharp**

```bash
npm install --save-dev sharp
```

Expected: `sharp` added to `devDependencies` in `package.json`.

- [ ] **Step 2: Convert SVG to 1024×1024 PNG**

```bash
node -e "
const sharp = require('sharp');
const fs = require('fs');
const svg = fs.readFileSync('build/icon.svg');
sharp(svg, { density: 300 })
  .resize(1024, 1024)
  .png()
  .toFile('build/icon.png')
  .then(() => console.log('build/icon.png written'))
  .catch(e => { console.error(e); process.exit(1); });
"
```

Expected output: `build/icon.png written`

Verify file exists and is non-zero:

```bash
ls -lh build/icon.png
```

Expected: a file around 20–100 KB.

- [ ] **Step 3: Commit**

```bash
git add build/icon.png package.json package-lock.json
git commit -m "feat: add 1024x1024 PNG app icon for electron-builder"
```

### Task 3: Verify the icon appears at runtime

**Files:**
- No changes — read-only verification step.

- [ ] **Step 1: Launch the app**

```bash
npm start
```

- [ ] **Step 2: Check the Dock (macOS)**

Look at the macOS Dock while the app is running. The icon should show the circular recovery arrow with a solid play triangle on a dark slate background instead of the default Electron icon.

If the Dock still shows the default Electron icon, quit and relaunch — macOS caches Dock icons and a cold start is needed after changing icon files.

- [ ] **Step 3: Confirm electron-builder picks it up**

Run a dry-run build to confirm no icon-related errors:

```bash
npm run build:mac -- --dir
```

The `--dir` flag skips packaging to a `.dmg` and just outputs the `.app` bundle to `dist/`. Check the result:

```bash
ls dist/mac*/TeslaCam\ Recovery.app/Contents/Resources/
```

Expected: an `electron.icns` or `icon.icns` file present.

- [ ] **Step 4: Commit if any config was adjusted**

If no config changes were needed (expected), no commit required. If `package.json` `build` section needed an explicit `icon` path, commit that change:

```bash
git add package.json
git commit -m "chore: set explicit icon path in electron-builder config"
```
